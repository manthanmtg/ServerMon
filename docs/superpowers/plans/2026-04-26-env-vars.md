# EnvVars Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a stateless, OS-aware EnvVars module that manages real host user environment variables and exposes system-scope instructions.

**Architecture:** Core env detection, parsing, quoting, and mutation lives in `src/lib/env-vars/service.ts` with focused unit tests. API routes require admin auth and delegate to the service. The module UI consumes the API, masks secrets by default, and explains restart/new-terminal behavior.

**Tech Stack:** Next.js App Router, TypeScript, Vitest, React, Tailwind CSS, lucide-react.

---

## File Structure

- `src/lib/env-vars/service.ts`: host OS detection, shell target selection, current env snapshot, shell env parsing, add/delete helpers, system command generation.
- `src/lib/env-vars/service.test.ts`: pure service behavior tests.
- `src/modules/env-vars/types.ts`: shared API and UI types.
- `src/modules/env-vars/module.ts`: module registration.
- `src/modules/env-vars/module.test.ts`: module registration tests.
- `src/modules/env-vars/ui/EnvVarsPage.tsx`: full management page.
- `src/modules/env-vars/ui/EnvVarsPage.test.tsx`: UI tests for list, mask toggle, add/delete.
- `src/modules/env-vars/ui/EnvVarsWidget.tsx`: dashboard widget summary.
- `src/modules/env-vars/ui/EnvVarsWidget.test.tsx`: widget tests.
- `src/app/api/modules/env-vars/route.ts`: GET, POST, DELETE handlers.
- `src/app/api/modules/env-vars/route.test.ts`: auth, validation, and service delegation tests.
- `src/app/env-vars/page.tsx`: route page wrapper.
- `src/app/env-vars/page.test.tsx`: route smoke test.
- `src/components/modules/ModuleWidgetRegistry.tsx`: widget registration.
- `src/components/layout/ProShell.tsx`: navigation entry.
- `src/modules/index.ts`: core module registration.
- `CLAUDE.md`: workspace index update.
